# 必有谱 AIGC 系统（Biupu Pro）使用说明

本文档面向开源用户与运维/管理员，说明系统**前端（用户端）**与**后端（管理端）**的功能与常见操作，便于在开源代码平台上查阅与二次开发参考。

---

## 一、系统概述

**必有谱 AIGC 系统** 是一套前后端分离的 Web 应用，主要包含：

| 端 | 说明 | 典型访问地址 |
|----|------|--------------|
| **用户端（前端）** | Vue 3 构建的 SPA，供访客/用户使用：打谱、有谱、AI 对话、个人中心等 | `https://你的域名/web/` 或独立部署的 dist |
| **管理端（后台）** | Vue 3 非构建模式，供管理员使用：配置、用户、订单、曲谱、激活码等 | `https://你的域名/admin/index.html` |

- **后端**：PHP 8.1+ / ThinkPHP 8，模块化设计（idapu、market、user、ai、chat、document 等），RESTful API，JWT 鉴权。
- **用户端** 支持 PC 与移动端自适应；**管理端** 以 PC 为主。

## 二、前端（用户端）功能说明

### 2.1 首页

- 入口为站点根路径（如 `/` 或 `/web/#/`）。
- 展示站点名称、Logo、导航等，可配置（后台「配置 → 系统配置」）。
- 导航可进入：**打谱**、**有谱**、**AI识谱**、**登录/注册**、**个人中心** 等。

### 2.2 打谱（iDapu 乐谱与 AI 识谱）

**路径**：`/notation/index`（新建）或 `/notation/:id`（编辑已有曲谱）

- **能力概览**
  - **双输入模式**：支持 **iDapu 文本** 与 **AlphaTex 文本** 两种乐谱格式，可在编辑器中切换；默认类型与示例内容可由后台「打谱配置」设置。
  - **实时渲染**：集成 AlphaTab，根据当前文本实时渲染六线谱/简谱/五线谱，支持播放、调速等。
  - **AI 打谱**：根据自然语言描述、歌词或和弦信息，由 AI 生成符合 AlphaTex 语法的乐谱文本，可直接在编辑器中继续修改并渲染。
  - **曲谱管理**：登录后可保存曲谱到「我的曲谱」、新建/编辑/删除，未登录部分能力会引导登录或激活码。

- **与激活码的关系**
  - 部分能力（如 AI 打谱、保存到云端）可能要求**已绑定有效激活码**；未激活时操作会提示「请先激活」或「购买激活码」。
  - 用户可在打谱页或个人中心入口**购买激活码**（若后台开启支付）、或**输入激活码**完成激活。

- **常见操作**
  - 新建曲谱：进入打谱页 → 选择类型（iDapu/AlphaTex）→ 编辑文本 → 保存（需登录）。
  - AI 打谱：在打谱页打开 AI 打谱面板 → 输入描述/歌词/和弦 → 生成后插入或替换当前内容。
  - 查看/编辑已有曲谱：从「我的曲谱」或直链 `/notation/:id` 进入。

### 2.3 有谱（市场 / 曲谱市场）

**路径**：`/market/index`

- **能力概览**
  - **曲谱浏览**：展示公开曲谱列表，支持按关键词搜索、按免费/会员专享筛选。
  - **曲谱详情与播放**：点击曲谱可查看详情；部分曲谱支持在线播放（AlphaTab 渲染），**会员专享**曲谱需登录且为有效会员方可播放/查看内容。
  - **会员与支付**：可在此入口**开通会员**（选择等级、微信/支付宝支付）或查看会员状态；支付方式是否可用由后台「系统配置」中微信/支付宝是否启用决定。

- **常见操作**
  - 浏览与搜索曲谱：进入有谱 → 使用搜索框或筛选条件。
  - 开通会员：点击「开通会员」→ 选择套餐与支付方式 → 完成支付后自动刷新会员状态。
  - 播放曲谱：在详情页点击播放（免费或已开通会员时可用）。

### 2.4 登录 / 注册 / 忘记密码

**路径**：`/auth/login`、`/auth/register`、`/auth/forget`

- 账号注册、登录、找回密码；可选短信/邮箱验证（依赖后台邮件或短信配置）。

### 2.5 个人中心（用户中心）

**路径**：`/user/index`，需登录。

- **个人中心首页**：汇总入口，如账户信息、激活码状态、会员状态、积分、签到、邀请等。
- **账户信息**（`/user/info`）：昵称、头像等资料修改。
- **充值套餐**（`/user/charge`）：若后台开启支付与充值相关配置，可在此查看套餐或充值（视具体业务而定）。
- **积分日志**（`/user/score_log`）：积分变动记录。
- **签到奖励**（`/user/award`）：每日签到与奖励（若后台开启）。
- **邀请好友**（`/user/invite`）：邀请链接与规则（若后台开启邀请）。

与**激活码 / 会员**相关的入口通常也在个人中心或顶部用户菜单中：

- **激活码**：查看「我的激活码」、输入新激活码激活、或跳转「购买激活码」。
- **会员**：查看会员到期时间、或跳转「开通会员」（有谱/市场）。

### 2.6 帮助文档

**路径**：`/document/help/:id`

- 展示后台「文档管理」中配置的帮助类文档，用于站点说明、使用教程等。

## 三、后端（管理端）功能说明

管理端访问地址一般为：`https://你的域名/admin/index.html`。登录后左侧为菜单，按模块与权限展示。

### 3.1 系统级（配置、用户、权限等）

- **配置**
  - **系统配置**：站点名称、Logo、备案号、支付总开关、关闭网站、允许复制、禁止访问 IP 等；**微信支付**与**支付宝支付**的开关及证书/密钥配置（商户号、AppID、证书、通知 URL 等）。
  - **附件设置**：上传平台（本地/腾讯云/阿里云/七牛云）、附件域名、允许的格式与大小等。
  - **邮件设置**：SMTP 等，用于注册/找回密码等邮件发送。
  - **短信设置**：短信平台与密钥，用于验证码等（若启用）。
- **用户管理**：用户列表、编辑、禁用等。
- **角色与权限**：角色管理、权限分配、菜单与按钮级权限。
- **模块管理**：模块的安装、卸载、启用、禁用等（若开放）。
- **总控 / 实例**：若启用总控，可查看实例上报与列表（多实例部署场景）。

### 3.2 iDapu 模块（打谱与激活码）

- **打谱配置**（`idapu/config`）
  - 默认输入模式、默认曲谱类型、默认曲谱内容、激活码单价、是否打开上次曲谱等；AI 打谱相关提示词与通道配置（通常仅管理员可见，避免敏感信息泄露）。
- **曲谱管理**（`idapu/score`）
  - 用户提交的曲谱列表，支持查看、编辑、删除、公开状态等。
- **购买订单**（`idapu/order`）
  - 激活码购买订单列表；顶部有统计概览（已开通会员、订单总数、已支付、待支付、订单总额）；操作列支持「查看」详情。
- **激活码管理**（`idapu/license` 或 `idapu/activation-code`）
  - 激活码的创建、批量生成、列表、编辑、删除；支持将某激活码「授权给用户」。
- **节奏型管理**（`idapu/rhythm-pattern`）
  - 节奏型数据，供前端 AI 识谱或打谱相关功能使用。

### 3.3 市场模块（有谱、会员、订单）

- **默认配置**（`market/config`）
  - 市场首页默认曲谱、是否打开默认曲谱等。
- **会员等级**（`market/vip_level`）
  - 会员等级的新增、编辑、删除；名称、价格、有效天数、排序、上架状态等。
- **订单管理**（`market/order`）
  - 会员开通订单列表；顶部有统计概览（曲谱数、今日订单、订单总数、已支付、待支付、订单总金额）；操作列支持「查看」详情。
- **谱子管理**（`market/score`）
  - 市场侧曲谱的上下架、公开范围等（与 idapu 曲谱可有关联或同步，视实现而定）。

### 3.4 用户模块

- **用户管理**（`user/user`）：见上文「系统级」。
- **参数设置**（`user/config`）：用户相关开关与规则，如邀请、签到、积分等。
- **积分日志**（`user/score_log`）：用户积分变动记录。
- **VIP 日志**（`user/vip_log`）：会员开通/续费等相关日志。

### 3.5 AI 模块

- **AI 通道**（`ai/channel`）：对接各 AI 平台的通道配置（API 地址、密钥等）。
- **Key 池**（`ai/key`）：API Key 池管理，供通道调用。

### 3.6 文档模块

- **文档管理**（`document/document`）：帮助文档、说明页等内容的创建与编辑，供前端「帮助文档」展示。

## 四、常见业务流程简述

### 4.1 用户使用「打谱」并购买激活码

1. 用户访问用户端 → 进入「打谱」。
2. 若需 AI 打谱或保存到云端，系统提示需激活码时，用户可选择「购买激活码」或「输入激活码」。
3. **购买激活码**：选择支付方式（微信/支付宝，依后台开关）→ 支付成功 → 自动发放激活码并刷新状态。
4. **输入激活码**：在弹窗中输入管理员发放的激活码 → 激活成功后即可使用需激活的功能。

### 4.2 用户在有谱「开通会员」

1. 用户进入「有谱」→ 点击「开通会员」或从个人中心进入。
2. 选择会员等级（套餐）与支付方式（微信/支付宝，依后台开关）。
3. 完成支付后，会员状态自动更新，可播放会员专享曲谱。

### 4.3 管理员发放激活码

1. 后台进入 **iDapu → 激活码管理**。
2. 可「批量生成」激活码，或对已有激活码执行「授权给用户」指定到某用户，用户端登录后即可使用。

### 4.4 管理员配置支付（微信/支付宝）

1. 后台 **配置 → 系统配置** 中开启「启用支付」。
2. 在 **微信支付** / **支付宝支付** 中填写商户号、AppID、证书、密钥及支付结果通知 URL；支付宝需配置应用公钥证书与根证书（详见项目内支付分析文档）。
3. 保存后，用户端「购买激活码」「开通会员」等页面的对应支付方式才会可用。

## 五、部署与入口汇总

| 用途 | 说明 |
|------|------|
| 用户端地址 | 将 `front-end/web/dist` 部署到 Web 目录，或配置 Nginx 指向该目录；常见路径如 `/web/` 或独立域名。 |
| 管理端地址 | 一般为 `https://域名/admin/index.html`，无需单独构建。 |
| 后端 API | 网站根目录指向 `public`，ThinkPHP 单入口；API 前缀依路由配置（如 `/idapu/index/`、`/market/index/`、`/passport/` 等）。 |

更详细的安装、环境要求与目录结构见项目根目录 **README.md** 与 **docs/INSTALL.md**。
